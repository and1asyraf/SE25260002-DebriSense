7.0 Deployment Overview

The DebriSense platform, developed using a Python Flask backend integrated with a dynamic HTML, CSS, and JavaScript frontend, is hosted on Vercel. By leveraging Vercel's serverless architecture, the application achieves efficient deployment, high availability, and seamless scalability. Vercel’s native integration with Git enables a streamlined Continuous Integration and Continuous Deployment (CI/CD) pipeline. The project's codebase is stored in a Git repository and directly linked to the Vercel dashboard. Once configured, Vercel automatically triggers a new deployment whenever code changes are pushed to the designated production branch, typically the main branch.

This deployment process begins with Vercel pulling the latest codebase from the repository. The platform automatically detects the Python environment, installs the necessary backend dependencies specified in the `requirements.txt` file, and configures the routing using the `vercel.json` file to run the Flask application as serverless functions. Simultaneously, the frontend static assets are processed and deployed to Vercel’s global Edge Network (CDN). This dual-layered deployment ensures fast load times by caching static content at edge locations worldwide while dynamically executing backend logic on demand.

The deployment workflow also incorporates automated developer features like branch previews. When commits are pushed to non-main branches or pull requests are opened, Vercel generates unique, temporary preview URLs. This allows the team to rigorously test new features, such as predictive model updates or new interactive map filters, in an isolated production-like environment before merging them into the live system. If a production deployment encounters an issue, Vercel retains a complete history of all previous deployments, enabling the team to execute an instant rollback to a stable version with a single click. Furthermore, Vercel provides robust support for environment variables. Sensitive configuration details—such as external Weather API keys, Gemini AI credentials, and database connection strings—are securely managed through the Vercel dashboard and safely injected into the serverless functions during both the build process and runtime execution.
